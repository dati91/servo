Functions

Function names are linked to the Web Bluetooth specification.

Columns:

  1. Step number from the specification.
  2. Name of the used HTML test for the step.
  3. Values, which we used as arguments for the DOM functions.
  4. Expected results (described in spec).
  5. What our implementation returns.

Notation

  • Green: The expected behavior and the result matches for that step.
  • Red: Testable, but not imlemented yet.
  • Blue: Testable, but not working correctly.
  • Yellow: Not testable step.

Tests

For each test case a button is created. By pressing these buttons, a test will run with the corresponding input values.
